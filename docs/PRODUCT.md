# JejuFlow 제품 명세

> 현재 기준: Flutter 앱  
> 구현 기준 문서: `docs/AI_IMPLEMENTATION_BRIEF.md`  
> 디자인 기준 문서: `docs/designreference/stitch_jejuflow_tourism_app/evergreen_modern/DESIGN.md`

---

## 1. 제품 정의

JejuFlow는 렌터카 없이 제주를 여행하는 국내외 관광객에게 **현재 날씨와 대중교통 조건에 맞는 관광지**를 추천하는 제주 특화 관광 안내 앱이다.

핵심 질문:

> 지금 날씨에, 지금 버스로, 지금 당장 어디 가면 돼?

---

## 2. 해결하려는 문제

제주 여행자는 관광지 정보, 날씨 정보, 버스 정보를 각각 다른 서비스에서 확인해야 한다. 특히 렌터카를 이용하지 않는 여행자는 정류장 위치, 버스 대기시간, 도보 거리, 날씨 적합성을 직접 비교해야 해서 여행 계획이 어렵다.

JejuFlow는 이 정보를 한 화면에서 결합한다.

| 문제 | JejuFlow의 답 |
|---|---|
| 날씨가 바뀌면 어디를 가야 할지 모름 | 비/강풍 시 실내 관광지를 우선 추천 |
| 버스로 갈 수 있는 관광지를 찾기 어려움 | 인근 정류장과 버스 대기시간을 관광지 카드에 표시 |
| 외국인이 제주 교통 정보를 이해하기 어려움 | 영어 중심 UI와 다국어 확장 구조 제공 |
| 관광지 앱과 교통 앱을 오가야 함 | 관광정보, 날씨, 교통을 하나의 추천으로 통합 |

---

## 3. 사용자

### 주 타깃

- 제주를 방문한 외국인 개별 관광객
- 렌터카 없이 이동하는 여행자
- 영어, 일본어, 중국어 안내가 필요한 사용자

### 보조 타깃

- 렌터카 비용이나 운전 부담 때문에 대중교통을 선택한 국내 여행자
- 1인 여행자
- 날씨 변화에 맞춰 일정을 바꿔야 하는 여행자

---

## 4. 앱 구조

현재 앱은 3탭 구조다.

| 탭 | 역할 |
|---|---|
| Now | 현재 날씨와 지역 기준으로 지금 갈 수 있는 관광지 추천 |
| Move | 관광지 탐색, 상세정보 확인, 대중교통 경로 확인 |
| Routes | 저장한 경로 관리, 날씨 영향 경고, 즉시 사용 |

---

## 5. 핵심 기능

### 5.1 날씨 기반 추천

- 기상청 단기예보를 사용해 제주시/서귀포시 날씨를 확인한다.
- clear/cloudy 상태에서는 야외 관광지를 포함한다.
- rain/windy/storm 상태에서는 indoor 또는 both 관광지를 우선한다.
- 날씨가 좋지 않을 때 outdoor 관광지를 선택하면 대체 실내 관광지를 제안한다.

### 5.2 대중교통 접근성 표시

- 관광지별 인근 정류장, 버스 번호, 대기시간, 도보시간을 표시한다.
- TAGO 버스도착정보를 사용한다.
- API 실패 시 정적 fallback 시간과 캐시를 사용해 앱이 비어 보이지 않게 한다.

### 5.3 관광지 정보 제공

- 한국관광공사 TourAPI로 제주 관광지 목록, 이미지, 상세정보, 영문 설명을 가져온다.
- 정적 데이터 `lib/core/constants/spot_data.dart`는 주요 관광지 fallback으로 유지한다.
- 관광지 카드는 이미지가 있으면 이미지를, 없으면 관광지별 accent color를 사용한다.

### 5.4 저장 경로 관리

- 사용자는 관광지 경로를 저장할 수 있다.
- 저장된 경로는 `Hive`에 보관한다.
- 저장된 경로가 현재 bad weather 영향을 받으면 Routes 탭에서 경고를 표시한다.

### 5.5 다국어 확장

- MVP는 영어 UI 중심으로 구현한다.
- 이후 한국어, 일본어, 중국어를 Settings 또는 localization 구조로 확장한다.
- 다국어 대상은 탭명, 날씨 상태, 관광지 설명, 정류장명, 경로 안내, 오류 문구다.

---

## 6. MVP 완료 기준

- 앱 실행 시 Now 탭에 현재 날씨와 추천 관광지 TOP 3가 표시된다.
- 날씨가 bad일 때 실내 관광지가 우선 추천된다.
- 관광지를 선택하면 Move 탭에서 경로 step을 볼 수 있다.
- 버스 대기시간, 도보시간, 총 이동시간이 표시된다.
- 경로를 저장하고 Routes 탭에서 다시 사용할 수 있다.
- 저장된 outdoor 경로가 현재 bad weather에 영향을 받으면 경고가 표시된다.
- TourAPI, 기상청, TAGO 중 일부가 실패해도 앱 전체가 중단되지 않는다.
- Evergreen Modern 디자인 방향이 주요 화면에 반영된다.

## 7. 향후 발전 방향

### Phase 1. MVP 안정화

- Now, Move, Routes 탭 완성
- API fallback 및 캐시 강화
- 위치 권한 거부 상태 처리
- Flutter analyze 통과

### Phase 2. 관광 추천 고도화

- 연관 관광지 추천
- 방문자 집중률 기반 혼잡도 badge
- 이미지 색상 추출 기반 다이나믹 컬러
- 실내 대체 코스 추천

### Phase 3. 여행 편의 기능 확장

- 다국어 UI
- 반나절/하루 코스 추천
- 지도 기반 도보 이동 안내
- 오프라인 캐시

### Phase 4. 출시 준비

- 개인정보 처리방침
- 위치 권한 문구 정리
- 앱 아이콘과 스크린샷
- API 키 프록시 또는 플랫폼 제한
- App Store/TestFlight 배포
